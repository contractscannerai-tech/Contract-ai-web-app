## 💳 DODO PAYMENT INTEGRATION (CRITICAL)

Use environment variables:

- DODO_TEAM_PLAN_PRODUCT_ID
- DODO_TEAM_PLAN_PRICE_ID
- DODO_API_KEY (if required)

---

### PURCHASE FLOW:

When user clicks "Upgrade to Team Plan":

- Call backend endpoint:
  POST /api/dodo/create-subscription

- Backend must:
  - Use DODO_TEAM_PLAN_PRICE_ID
  - Create subscription via Dodo API
  - Return checkout/payment link

---

### PAYMENT CONFIRMATION (WEBHOOK)

Create endpoint:

POST /api/dodo/webhook

Dodo will notify backend when:

- payment successful
- subscription active
- subscription canceled
- payment failed

---

### WHEN PAYMENT IS SUCCESSFUL:

Backend MUST:

- Create team
- Assign user as OWNER
- Set:
  - plan = "team"
  - maxMembers = 5
  - full feature access

---

### WHEN SUBSCRIPTION FAILS OR IS CANCELED:

Backend MUST:

- Downgrade ALL team members to Free plan
- Remove team access
- Deactivate or delete team
